開発者向けドキュメント

						     Last Change: 24-Jan-2005.

* ディレクトリ構成
  + bdf.d	スケルトン作成用BDF(M+ bitmap 2.1.4より)
  + doc		ユーザは読まなくて済むドキュメント
  + eps.d	複合EPS
  + ucstable.d	文字コード変換用テーブル
  + work.d	一時的に生成するEPS

* eps.d内のファイル
  + */codemap
    あるファイルがどの文字のデータを持っているかを16進文字コードで示すファイ
    ル。1行1ファイルで複数の文字を含むことができ、#で始まる行はコメントとみな
    し及び空行は無視される。
  + checkall.sh
    全EPSの改行コードをチェックする。
  + commit.sh
    改行コードをチェックした後にcvs commitを実行する。
  + datadirs
    EPSデータの入っているサブディレクトリを指定するファイル。1行1ディレクトリ
    でコメントを入れることはできない。
  + eolcheck.pl
    指定したファイルの改行コードをLFに統一する。

* 注意
  scripts/build_skelton.shは非常に重い。生成されるファイル数が非常に多いのに加
  えて、sfdの生成にいたってはメモリも馬鹿食い、P4-3.2GHzでも10分近くを要する。
  実行にはくれぐれも注意のこと。
